# Mobile readiness — RM surface tracker

> **Living checklist.** Built from the 2026-07-15 static audit of every regional-manager
> page/component at a phone viewport (~375–430px). 103 findings: 11 high / 50 medium /
> 42 low. Tick items as they land; add rows when new issues surface. The same recipe
> rolls out to Supplier / SM / Admin after the RM pass is approved.
>
> **Method (applies to every fix):** mobile-first additive — base classes change, `sm:`/`lg:`
> restore today's exact desktop rendering, so web is pixel-identical and only the phone
> needs re-checking. One phase per commit batch; owner checks on a real phone per phase.

## The four standing mobile rules

1. **Primary lists never scroll horizontally.** A wide grid/table gets a stacked-card
   fallback under `sm` (`sm:hidden` cards + `hidden sm:block` table). `overflow-x-auto`
   is only for secondary/document content (e.g. print-style report tables), never the
   main working list.
2. **Badge clusters never starve primary text.** No rigid `shrink-0` rows of fixed-width
   badges beside a title at base — stack them (`flex-col items-end … sm:flex-row`) or
   un-fix the width (`w-auto sm:w-[120px]`).
3. **All pop-ups use the shared bottom-sheet `Modal`** (`components/ui/Modal.tsx`) — no
   hand-rolled centered overlays (they get cramped `p-4` gutters at 375px).
4. **Mobile density scale:** cards `p-4`, icon chips `h-10`, key values `text-xl`,
   labels may `line-clamp-2` — with `sm:` restoring the desktop scale.

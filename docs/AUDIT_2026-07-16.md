## 1. Security & authorisation

The external audit found 3 real issues (X1‚ÄìX3). Our deeper pass found 4 more (S1‚ÄìS4), three of them worse than anything ChatGPT flagged. All 7 are **fixed on this branch**. Root cause was the same everywhere: the app's threat model treated everyone inside a company as trusted, but **suppliers are competing outsiders inside the same tenant** ‚Äî and two pages predating v3 had no scoping at all.

- [x] **X1 ‚Äî Supplier store page leaked all store tickets + competitors' quote amounts** (HIGH)
  `app/supplier/stores/[id]/page.tsx` fetched every ticket for the store with embedded `quotes(amount)`, gated only by same-company. Any supplier could read rival suppliers' quote amounts.
  **Fix (76d688e):** tickets filtered to awarded (`tickets.supplier_id`) or invited (`ticket_suppliers`) for the caller's `supplier_users` orgs ‚Äî mirrors the ticket-detail gate; quotes embed dropped.
- [x] **X2 ‚Äî Any supplier could reassign / wipe regional managers** (HIGH)
  `POST /api/supplier/assign-rm` let any supplier-role user upsert an RM onto a region, and its "clear" branch deleted **every** `regional_users` row for the region. Admin-era leftover; RM linking belongs to the system-admin Hierarchy tab only.
  **Fix (6c754ae):** route + `AssignRMForm` deleted; supplier store page stripped.
- [x] **X3 ‚Äî Verification-doc reference check was `url.includes('supplier-docs')`** (MEDIUM)
  A crafted reference (any string containing `supplier-docs`) was stored and later **signed with the service-role client** on read-back. Exploitability low (paths contain user-UUID + timestamp + random), but real.
  **Fix (76d688e):** reference must parse (`bucketAndPath`) to bucket `supplier-docs` AND path prefix `${user.id}/`; `supplier-docs` added to `STORAGE_BUCKETS`.
- [x] **S1 ‚Äî `/supplier/regional` listed every RM on the platform, ALL tenants** (HIGH)
  Admin-client query on `user_profiles` with `role='regional_manager'` and **no company filter** ‚Äî full name/company/email/phone of every regional manager across every company, readable by any supplier including unverified self-signups. Orphaned page (not in nav).
  **Fix (89d0cb3):** page deleted.
- [x] **S2 ‚Äî `/supplier/regional/[id]` ‚Äî zero auth, any RM's contacts + full store estate** (HIGH)
  No `auth.getUser`, no company check ‚Äî walk any user-id, get that RM's email/phone/address and every store (with physical addresses) in their regions.
  **Fix (89d0cb3):** page deleted.
- [x] **S3 ‚Äî `GET /api/suppliers` returned every supplier row across all tenants** (HIGH)
  `select('*')` with no tenant filter (email, phone, VAT number, verification status). Dead route ‚Äî its UI was deleted in `4a143c9` ‚Äî but live and reachable. Siblings POST/bulk inserted rows with no `company_id` (MEDIUM).
  **Fix (89d0cb3):** all three `/api/suppliers*` routes deleted.
- [x] **S4 ‚Äî Invited supplier could award themselves the ticket** (MEDIUM)
  `/api/tickets/[id]/transition` wrote `body.supplierId` into `tickets.supplier_id` for `require_assessment`/`request_quote` with no validation, and `hasAccess()` passes on a mere invite ‚Äî an invited supplier could set `supplier_id` to themselves, flipping every "awarded" check (RM‚Üîsupplier chat, awarded-only surfaces) and pre-empting the competitive award.
  **Fix (89d0cb3):** `supplierId` is ignored for supplier-role callers; internal roles unaffected.

**Verified solid** (adversarially checked, no action needed): storage path confinement + atomic upload quota; WhatsApp webhook HMAC fail-closed; cron auth; supplier-scoped queries everywhere else (`.in('supplier_id', supplierIds)` pattern); `rmOwnsTicket()` on all RM mutations; quote-decision idempotency (no double-award); role escalation via `PATCH /api/profile` explicitly blocked; project routes fully tenant-scoped; admin routes re-check `system_admin` server-side; forgot-password doesn't leak account existence; zod validation on effectively all write routes; rate limits everywhere that matters.

**Still open:**
- [ ] Independent penetration test (owner-track; after these fixes, not before)
- [ ] E2E role tests that would have caught S1‚ÄìS4 (see ¬ß4)
- [ ] `pdfjs-dist` worker loads from unpkg CDN at runtime (`components/admin/SendQuoteForm.tsx`) ‚Äî external script dependency; consider self-hosting the worker (LOW)

**Score: 8.0** ‚Äî the tenant/role model is now consistently enforced; what remains is external validation, not known holes.
